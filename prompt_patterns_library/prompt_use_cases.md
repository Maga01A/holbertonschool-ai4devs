1. Code Quality & Refactoring
Complexity Reduction

Goal: Break down long, complex functions into smaller, manageable ones.

Input: A legacy code block (100+ lines).

Output: Modularized functions following the Single Responsibility Principle.

Language Migration

Goal: Convert a code snippet from one language to another while maintaining logic.

Input: Python script.

Output: Equivalent, idiomatic JavaScript (Node.js) code.

Performance Optimization

Goal: Identify and fix bottleneck operations (e.g., nested loops).

Input: A function with high time complexity.

Output: Optimized code using better algorithms or data structures.

2. Debugging & Error Handling
Stack Trace Analysis

Goal: Interpret cryptic error logs and find the root cause.

Input: Error message + relevant code snippet.

Output: Explanation of the error and a step-by-step fix.

Edge Case Identification

Goal: Find scenarios where the code might fail (null values, empty lists).

Input: A functional code block.

Output: A list of potential edge cases and defensive programming suggestions.

3. Documentation & Knowledge Sharing
API Documentation Generation

Goal: Create standardized documentation for backend endpoints.

Input: Controller or Route code.

Output: Swagger/OpenAPI formatted documentation.

Code Explanation for Juniors

Goal: Explain complex logic in plain language for onboarding.

Input: Advanced regex or multi-threaded logic.

Output: Line-by-line breakdown using analogies.

4. Testing & Validation
Unit Test Generation

Goal: Automate the creation of test suites for functions.

Input: A Python or JS function.

Output: Test cases using PyTest or Jest, covering both happy and sad paths.

Mock Data Creation

Goal: Generate realistic JSON data for frontend testing.

Input: A database schema or interface definition.

Output: A JSON array with 10+ realistic entries.

Integration Test Scenarios

Goal: Outline how different modules should interact during a test.

Input: System architecture description.

Output: A sequence of test steps for end-to-end validation.
